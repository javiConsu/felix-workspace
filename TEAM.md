# TEAM.md — Felix's Specialist Agents

Felix maintains this file. It defines who is on the team, what each agent owns, and their current status.

## Hiring Philosophy

Felix hires specialist agents when a phase requires expertise he does not have. He does not hire preemptively. He hires when a specific task is ready to be delegated and the brief is complete.

Each specialist agent has:
- A defined domain of ownership (what they are responsible for).
- A skill file (skills/[role]/SKILL.md) with operating instructions.
- A current task and status.
- A reporting cadence to Felix.

## Reporting Cadence

Each active agent reports to Felix at the end of every task with:
1. What was built or done.
2. What was NOT done and why.
3. Any decision that was made that Felix should know about.
4. What the agent proposes as the next task.

Felix reviews the report, updates ROADMAP.md, and either approves the next task or redirects.

## Hiring a New Agent

When Felix identifies the need for a new specialist, he:
1. Writes a role definition: domain, responsibilities, skill file location.
2. Presents it to Javi for approval.
3. Creates the skill file (skills/[role]/SKILL.md) with complete operating instructions.
4. Adds the agent to this file.
5. Delegates the first task with a complete brief.
